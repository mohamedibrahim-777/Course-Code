ABSTRACT

In the modern era of digital education, providing a structured and accessible platform for learning programming languages is essential for students and aspiring developers. Traditional learning methods such as textbook-based teaching and unstructured online resources often lack interactivity, progress tracking, and institutional oversight.

Course Code is a comprehensive web-based Programming Learning Platform designed to address these challenges. The platform offers structured programming courses with organized modules, video tutorials, downloadable study materials, and real-time progress tracking. It supports a role-based management system with three distinct user roles: Head of Department (HOD), Staff, and Students.

Staff members can create and manage courses across 18+ programming languages (Python, Java, JavaScript, C++, SQL, Go, Rust, etc.), add lessons with video content and downloadable resources (PDFs, documents, Excel sheets, images), and monitor student progress through an analytics dashboard. The HOD has a comprehensive analytics dashboard for department-wide performance monitoring, including staff performance tracking, student activity analysis, and course statistics.

Students can browse available courses, view lessons with embedded video content, download study materials, and track their learning progress. The platform includes an engagement tracking system that requires students to spend minimum reading time, watch videos, and download resources before marking lessons as complete, ensuring genuine learning engagement.

The platform is built using modern web technologies including React.js with TypeScript for the frontend, Express.js for the backend API server, SQLite (via better-sqlite3) for data storage, Tailwind CSS for responsive styling, and JWT-based authentication for secure access control. An OTP-based registration system ensures verified student sign-ups.

Keywords: Programming Learning Platform, E-Learning, Role-Based Access Control, Progress Tracking, React.js, Express.js, SQLite, JWT Authentication

1. INTRODUCTION

1.1 Introduction to Project

In the rapidly evolving landscape of technology, programming skills have become indispensable for students across all engineering disciplines. The demand for structured, accessible, and institution-managed learning platforms has grown significantly as traditional classroom teaching alone cannot keep pace with the diverse programming languages and frameworks required in the industry today.

The proposed project, Course Code, is a comprehensive web-based Programming Learning Platform designed to provide students with a structured pathway to learn programming languages. Unlike generic online learning platforms, Course Code is designed with an institutional hierarchy in mind, featuring role-based access for the Head of Department (HOD), Staff members, and Students.

The platform enables staff members to create programming courses covering 18+ languages including Python, Java, JavaScript, C++, SQL, Go, and Rust. Each course contains structured lessons with rich text content, embedded video tutorials, and downloadable study materials such as PDFs, documents, and Excel sheets. Students can enroll in courses, progress through lessons at their own pace, and track their learning journey through an intuitive dashboard.

A distinctive feature of Course Code is its engagement tracking system. Students must genuinely engage with the content — spending a minimum reading time of 30 seconds per lesson, watching embedded videos, and downloading associated resources — before they can mark a lesson as completed. This ensures authentic learning rather than superficial progression.

The HOD dashboard provides department-wide analytics, including staff performance metrics, student activity reports, and course distribution statistics. This enables informed decision-making about curriculum improvements and teaching effectiveness.

Built using modern technologies such as React.js with TypeScript, Express.js, SQLite, Tailwind CSS, and Framer Motion for animations, Course Code delivers a polished, responsive, and performant user experience. JWT-based authentication with OTP verification ensures secure access to the platform.

1.2 Project Category

The Course Code project falls under the category of Web-Based Educational Learning Management System (LMS). It is an e-learning platform that integrates functionalities such as course management, lesson delivery, progress tracking, resource distribution, user role management, and analytics. The platform serves as a specialized programming education tool with department-level administrative oversight.

1.3 Objectives

The key objectives of this project are:

  • Structured Learning: Providing a well-organized platform with courses categorized by programming language and difficulty level (beginner, intermediate, advanced).
  • Multi-Role Management: Implementing a role-based system with HOD (analytics and oversight), Staff (course creation and management), and Student (learning and progress) roles.
  • Content Delivery: Enabling rich content delivery through text lessons, video tutorials (local and YouTube), and downloadable resources (PDF, documents, Excel, images).
  • Progress Tracking: Providing real-time progress tracking with engagement verification to ensure genuine learning.
  • Analytics Dashboard: Offering comprehensive analytics for staff and HOD to monitor student performance, course effectiveness, and department statistics.
  • Secure Authentication: Implementing JWT-based authentication with OTP verification for student registration.
  • Responsive Design: Ensuring the platform works seamlessly across desktops, tablets, and mobile devices.

1.4 Problem Statement

Traditional programming education systems face several challenges that hinder effective learning outcomes:

  • Lack of Structured Content: Students often rely on scattered resources from multiple websites, leading to an inconsistent learning experience without a clear pathway.
  • No Institutional Oversight: Most online platforms operate independently without any integration with educational institutions, preventing faculty from monitoring student progress.
  • Absence of Engagement Verification: Existing platforms allow students to skip through content and mark lessons as complete without genuine engagement.
  • Limited Resource Management: There is no centralized system for distributing study materials like PDFs, documents, and practice sheets alongside course content.
  • No Analytics for Decision-Making: Institutions lack data-driven insights about student performance and course effectiveness.
  • Poor User Experience: Many existing platforms have outdated interfaces that fail to engage modern students accustomed to polished web applications.

Course Code addresses all these challenges by providing a comprehensive, institution-managed programming learning platform with engagement tracking, resource management, and analytics capabilities.

Existing System:

Currently, programming education in institutions relies on:

  • Traditional classroom lectures with limited hands-on practice time.
  • Generic online platforms (Codecademy, freeCodeCamp, YouTube tutorials) without institutional integration.
  • Manual distribution of study materials through email or file-sharing services.
  • No systematic tracking of individual student progress in programming courses.

Demerits:

  • No Role-Based Access: Existing solutions do not provide institutional hierarchy-based access control.
  • Inconsistent Quality: Content quality varies widely across free online resources.
  • No Analytics: Institutions cannot make data-driven decisions about curriculum improvements.

3. DESIGN

3.1 System Analysis

Overview

System analysis involves examining the key functional and non-functional requirements of Course Code to ensure a well-structured and efficient system. The platform is designed to serve three distinct user groups with specific access levels and capabilities.

User Requirements

Students:

  • Register with OTP verification and create their profile.
  • Browse available programming courses categorized by language and difficulty level.
  • View lessons with text content, embedded videos, and downloadable resources.
  • Track their learning progress through a personalized dashboard.
  • Download study materials (PDFs, documents, Excel sheets, images).
  • Mark lessons as complete after fulfilling engagement requirements.
  • Update their profile information and change passwords.

Staff Members:

  • Create, edit, and delete programming courses.
  • Add, modify, and remove lessons within courses.
  • Upload video tutorials and study materials for each lesson.
  • Monitor individual student progress and completion rates.
  • View analytics summaries (total students, active courses, average progress).

Head of Department (HOD):

  • View department-wide analytics and statistics.
  • Monitor staff performance (courses created by each staff member).
  • Track student activity and lesson completion across all courses.
  • Generate reports for staff, students, and courses.
  • View course distribution by programming language.

Functional Requirements

  • User Authentication: Secure login using email/password with JWT token generation. OTP-based registration for new students.
  • Course Management: CRUD operations for courses with fields for title, description, language, and level.
  • Lesson Management: Ordered lessons within courses with text content, video URLs, and resource file uploads.
  • Progress Tracking: Record and display lesson completion status with engagement verification.
  • File Upload: Server-side file storage for videos and documents up to 50MB.
  • Analytics: Summary statistics and detailed reports for staff and HOD roles.
  • Profile Management: User profile update and password change functionality.

Non-Functional Requirements

  • Security: Passwords hashed with bcrypt (10 salt rounds), JWT tokens with 24-hour expiry, role-based API authorization.
  • Performance: Optimized SQLite queries, Vite-based fast builds, lazy loading of resources.
  • Usability: Clean, minimal UI with smooth animations using Framer Motion.
  • Reliability: Server-side validation, error handling, and database constraints.
  • Scalability: Modular architecture allowing easy addition of new features.

3.2 System Architecture

Course Code follows a two-tier client-server architecture where the Express.js server handles both the API and serves the frontend application.

1. Presentation Layer (Frontend)

  • Built with React.js 19 and TypeScript for type-safe component development.
  • Tailwind CSS 4 for utility-first responsive styling.
  • Framer Motion for page transitions and interactive animations.
  • React Router DOM for client-side routing with animated page transitions.
  • Context API (AuthContext) for global authentication state management.
  • Lucide React for consistent iconography throughout the application.

2. Business Logic Layer (Backend)

  • Express.js server handling RESTful API endpoints.
  • JWT middleware for authentication and authorization on protected routes.
  • Role-based access control (student, staff, hod) on API endpoints.
  • Multer middleware for multipart file upload processing.
  • Vite development server integration for hot module replacement during development.

3. Data Layer (Database)

  • SQLite database with better-sqlite3 for synchronous, high-performance queries.
  • Five tables: users, courses, lessons, progress, otps.
  • Foreign key relationships maintaining referential integrity.
  • Cascade deletion (courses → lessons → progress) for data consistency.

3.3 Work Flow Diagram

The following workflow outlines how users interact with Course Code:

1. User Registration & Login

New students register with name, email, and password → OTP is generated and displayed in server console → Student verifies OTP → Account is created → JWT token is issued. Existing users log in with email/password → JWT token is generated → User is redirected to role-appropriate dashboard.

2. Dashboard Navigation

Based on user role: HOD sees the Analytics Dashboard → Staff sees the Control Panel → Student sees the Learning Dashboard.

3. Course & Lesson Management (Staff)

Staff creates a new course (title, language, level, description) → Adds lessons with content, video, and resources → Uploads files through the file upload system → Monitors student progress.

4. Learning Flow (Student)

Student browses available courses → Selects a course → Views lesson content → Watches embedded video (if available) → Downloads study material (if available) → Spends minimum reading time → Marks lesson as complete → Progresses to next lesson.

5. Analytics & Reporting (HOD)

HOD views department stats → Monitors staff performance → Tracks student activity → Views course distribution → Generates detailed reports.

Level 0 Data Flow Diagram (Context Diagram)

The Level 0 DFD provides a high-level overview of data flow between users and the system:

  • Student (registers, logs in, browses courses, completes lessons)
  • Staff (creates courses, manages lessons, monitors progress)
  • HOD (views analytics, generates reports)

Central Process: Course Code Platform

  • Users Database (user profiles, credentials, roles)
  • Courses Database (course details, language, level)

  • Progress Database (completion records with timestamps)
  • OTP Store (temporary OTP records for registration)

Level 1 Data Flow Diagram

A detailed breakdown of how individual processes interact with data stores:

1. Authentication Process: User submits credentials → Server validates against Users database → JWT token generated → Token returned to client → Stored in localStorage.

2. Registration Process: Student submits registration details → OTP generated and stored in OTP Store → Student submits OTP → OTP verified and deleted → User record created in Users database → JWT token issued.

3. Course Management Process: Staff submits course details → Course record created/updated/deleted in Courses database → Associated lessons and progress records cascade-deleted on course deletion.

4. Lesson Management Process: Staff submits lesson details with optional file uploads → Files stored in server file storage → Lesson record created in Lessons database with file URLs.

5. Progress Tracking Process: Student completes engagement requirements → Completion record created in Progress database → Progress data aggregated for dashboard display.

6. Analytics Process: System queries Users, Courses, Lessons, and Progress databases → Aggregated statistics computed (totals, averages, distributions) → Results displayed on Staff/HOD dashboards.

  CREATE TABLE IF NOT EXISTS lessons (

    resource_type TEXT, -- 'pdf', 'excel', 'doc', 'image'

    FOREIGN KEY(course_id) REFERENCES courses(id)
  );

  CREATE TABLE IF NOT EXISTS progress (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    lesson_id INTEGER,
    completed_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY(user_id) REFERENCES users(id),
    FOREIGN KEY(lesson_id) REFERENCES lessons(id)
  );

5.1.2 JWT Authentication Middleware (server.ts)

const authenticate = (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1];
  if (!token) return res.status(401).json({ error: 'Unauthorized' });
  try {
    req.user = jwt.verify(token, JWT_SECRET);
    next();
  } catch (e) {
    res.status(401).json({ error: 'Invalid token' });
  }
};

5.1.3 Engagement Tracking Logic (CourseView.tsx)

const canMarkComplete = (lesson) => {
  if (completedLessons.includes(lesson.id))
    return { ready: true, reasons: [] };
  const reasons = [];
  const spent = timeSpent[lesson.id] || 0;
  if (spent < MIN_READ_SECONDS)
    reasons.push(`Read for ${MIN_READ_SECONDS - spent}s more`);
  if (lesson.video_url && !videoWatched[lesson.id])
    reasons.push('Watch the video');
  if (lesson.resource_url && !resourceDownloaded[lesson.id])
    reasons.push('Download the resource');
  return { ready: reasons.length === 0, reasons };
};

5.1.4 Auth Context Provider (AuthContext.tsx)

export const AuthProvider = ({ children }) => {
  const [user, setUser] = useState(null);
  const [token, setToken] = useState(null);

  useEffect(() => {
    const savedToken = localStorage.getItem('token');
    const savedUser = localStorage.getItem('user');
    if (savedToken && savedUser) {
      setToken(savedToken);
      setUser(JSON.parse(savedUser));
    }
  }, []);

  const login = (newToken, newUser) => {
    setToken(newToken);
    setUser(newUser);
    localStorage.setItem('token', newToken);
    localStorage.setItem('user', JSON.stringify(newUser));
  };

  const logout = () => {
    setToken(null); setUser(null);
    localStorage.removeItem('token');
    localStorage.removeItem('user');
  };

  return (
    <AuthContext.Provider value={{ user, token, login, logout }}>
      {children}
    </AuthContext.Provider>
  );
};

6. CONCLUSION

The Course Code Programming Learning Platform has been successfully designed and developed to provide a structured, institution-managed solution for programming education. The platform bridges the gap between traditional classroom teaching and modern e-learning by combining the benefits of both approaches — institutional oversight with the convenience and interactivity of web-based learning.

Key achievements of the project include:

  • Comprehensive Course Management: Staff can create and manage courses across 18+ programming languages with structured lessons containing text, video, and downloadable resources.
  • Engagement-Verified Progress Tracking: The innovative engagement tracking system ensures students genuinely interact with lesson content before marking completion, promoting authentic learning.
  • Role-Based Access Control: Three-tier user hierarchy (HOD, Staff, Student) with appropriate access levels and dashboards, enabling effective institutional management.
  • Data-Driven Analytics: HOD and Staff dashboards provide actionable insights about student performance, course effectiveness, and department statistics.
  • Secure Authentication: JWT-based authentication with OTP verification for registration and bcrypt password hashing ensures platform security.
  • Modern User Experience: Built with React.js, TypeScript, Tailwind CSS, and Framer Motion, the platform delivers a polished, responsive, and performant experience.
  • Resource Distribution: Centralized file upload and download system for study materials (PDFs, documents, Excel sheets, images, videos).

The project demonstrates the effective application of modern full-stack web development technologies in creating an educational platform that serves the needs of students, faculty, and department heads.
